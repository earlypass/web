# CLAUDE.md — earlypass-web

## What this repo is

Static marketing site and docs for EarlyPass. Built with Astro, deployed to GitHub Pages at `www.earlypass.app`.

The app itself (API, dashboard, widget, MCP server) lives in the separate `earlypass` repo and is served from `api.earlypass.app`.

## Key commands

```bash
npm install
npm run dev      # dev server at http://localhost:4321
npm run build    # build to dist/
npm run preview  # preview built output
```

## Structure

```
src/
├── layouts/
│   ├── BaseLayout.astro   # HTML shell, all CSS, header, footer
│   └── DocsLayout.astro   # BaseLayout + docs sidebar (activeDoc prop)
└── pages/
    ├── index.astro
    └── docs/
        ├── getting-started.astro
        ├── widget.astro
        ├── mcp.astro
        └── api.astro       # Full-height Scalar API reference
public/
├── CNAME                   # www.earlypass.app — required for GitHub Pages custom domain
├── og-image.png
└── robots.txt              # References sitemap-index.xml (generated by @astrojs/sitemap)
```

## Important URLs

All links pointing at the app must use `api.earlypass.app`, not `www.earlypass.app`:

| What | URL |
|------|-----|
| Dashboard login | `https://api.earlypass.app/dashboard/login` |
| Widget JS | `https://api.earlypass.app/widget/widget.js` |
| MCP endpoint | `https://api.earlypass.app/mcp` |
| OpenAPI spec | `https://api.earlypass.app/api/v1/openapi.yaml` |

Do not use relative paths for app links — this site and the app are on different origins.

## Adding a new page

1. Create `src/pages/<path>.astro`
2. Use `DocsLayout` for doc pages (pass `activeDoc` prop matching the sidebar link), `BaseLayout` for everything else
3. Add the route to `astro.config.mjs` sitemap if it should be indexed (the sitemap integration auto-discovers pages, so this is usually automatic)

## Curly braces in code blocks

Astro treats `{}` in templates as JS expressions. In `<pre><code>` blocks that contain literal curly braces, use `set:html` with a template literal:

```astro
<pre><code set:html={`GET /api/v1/things/{id}`}></code></pre>
```

Or escape with HTML entities: `&#123;` / `&#125;`.

## Deployment

Every push to `main` triggers `.github/workflows/deploy.yml`, which builds with `npm run build` and deploys `dist/` to GitHub Pages. No manual steps needed.
